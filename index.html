Why Modernise Your Windows Desktop Strategy?

The shift from traditional infrastructure to cloud-native endpoints

Windows 365 represents a fundamental shift in how organisations deliver Windows desktops. Rather than replicating traditional on-premises complexity in the cloud, the greatest benefits come from embracing cloud-native approaches that reduce infrastructure overhead while improving security and user experience.

The opportunity

Many organisations approaching Windows 365 attempt to mirror their existing VDI or on-premises infrastructure. While Windows 365 supports this approach, doing so means inheriting the same complexity, maintenance burden, and security challenges that cloud adoption should eliminate.

Accelerated Deployment

With Microsoft Hosted Network and gallery images, Cloud PCs can be provisioned in minutes rather than weeks. No Azure networking expertise required, no custom images to maintain, no domain controllers to configure.

Traditional: 2-4 weeks setup Modern: Same day deployment

Predictable Costs

Cloud PCs use a simple per-user, per-month pricing model. No variable compute costs, no storage management, no capacity planning. Finance teams can forecast with confidence.

Traditional: Variable infrastructure costs Modern: Fixed monthly per-user cost

Zero Trust Security

Entra ID Join with Conditional Access enables passwordless authentication, device compliance checks, and identity-based security—without exposing domain controllers or maintaining complex network perimeters.

Traditional: Network perimeter security Modern: Identity-based Zero Trust

Evergreen Updates

Windows Autopatch automates the entire update lifecycle—testing, deployment, and rollback. Microsoft manages the complexity of keeping Windows, Microsoft 365 Apps, Edge, and Teams current.

Traditional: Manual patching cycles Modern: Fully automated updates

Reduced IT Burden

Cloud-native management through Intune eliminates the need for SCCM infrastructure, Group Policy complexity, and on-premises management servers. One console for all endpoints.

Traditional: Multiple management tools Modern: Single Intune console

Data Protection

OneDrive with Known Folder Move ensures user data is automatically backed up, versioned, and accessible from any device. No file server infrastructure to maintain or back up.

Traditional: File server backups Modern: Automatic cloud sync

Our Recommendation

This guide is intentionally opinionated. While we'll help you configure Windows 365 to meet your specific requirements—including hybrid scenarios where truly necessary—we strongly advocate for the simplified, cloud-native approach wherever possible.

The "traditional" options exist for organisations with genuine constraints: regulatory requirements mandating domain membership, legacy applications requiring Kerberos authentication, or phased migration strategies. But for most organisations, the modern approach delivers better security, lower costs, and improved user experience.

When Hybrid is the right choice
  • Legacy apps requiring NTLM authentication
  • Strict Group Policy requirements
  • Regulatory compliance mandating domain join

No VNet, VPN Gateway, or ExpressRoute required. Microsoft manages all networking infrastructure.

Entra Internet Access for security

Add cloud-native Secure Web Gateway - web filtering, threat protection, M365 optimization without on-prem proxies.

Learn about Internet Access
RDP Shortpath supported

Enable public RDP Shortpath for direct UDP connectivity and reduced latency.

Additional cost and complexity

ANC requires Azure infrastructure and ongoing management. Consider if Microsoft Hosted Network with Global Secure Access could meet your needs.

Azure costs to consider
  • VPN Gateway: ~$140-350/month (depending on SKU)
  • ExpressRoute: $55-500+/month + circuit costs
  • Data egress charges for on-prem traffic
  • VNet itself is free, but peering has costs
Management overhead
  • Subnet IP address planning (/26 minimum)
  • DNS configuration and troubleshooting
  • VPN Gateway maintenance and monitoring
  • Network security group rules
  • ANC health check failures to diagnose
Longer setup time

VPN Gateway deployment takes 30-45 minutes. ExpressRoute provisioning can take days. ANC validation adds to provisioning time.

When ANC is the right choice
  • Hybrid Entra ID Join requirement
  • High-bandwidth on-prem data access
  • Regulatory requirement for VNet isolation
  • Existing Azure infrastructure to leverage

Select an image type

Recommended Approach

Gallery images are the fastest path to deployment. Microsoft maintains them with the latest updates and security patches.

Apps? Use Intune!

Don't bake apps into images - deploy them via Intune instead:

  • Centralized app management
  • Easy updates without rebuilding images
  • User self-service via Company Portal
  • Works with any gallery image
Provisioning time

Gallery images provision quickly (~30-45 min). Apps deployed via Intune install during or after provisioning.

Always current

Microsoft updates gallery images monthly with latest Windows updates - no maintenance required on your part.

Additional overhead

Custom images require ongoing maintenance. Consider if Intune app deployment could meet your needs instead.

Maintenance requirements
  • Monthly image updates recommended
  • Sysprep and generalize for each update
  • Upload to Azure Compute Gallery
  • Test before production deployment
When custom images make sense
  • Very large applications (10GB+ like AutoCAD, Visual Studio)
  • Complex app dependencies requiring specific install order
  • Strict compliance requiring validated base images
  • Specialized developer environments
Hybrid approach

Put large/complex apps in the image, use Intune for everything else. This balances provisioning speed with manageability.

Select a data storage strategy

Choose where users will store their documents and files.

Recommended Approach

OneDrive with Known Folder Move provides the best user experience for Cloud PCs - data follows the user everywhere.

Known Folder Move (KFM)

Automatically redirect Desktop, Documents, and Pictures to OneDrive:

  • Seamless user experience - no change in behavior
  • Data available on any device
  • Automatic backup and sync
  • Configure via Intune policy
Configure KFM
Files On-Demand

Files appear in File Explorer but download on access - saving Cloud PC disk space while providing full access to user's OneDrive.

Built-in protection

OneDrive includes version history (30 days), ransomware detection, and file recovery. Data is protected even if Cloud PC is compromised.

Transition strategy

Start with OneDrive for user profiles, gradually migrate team shares to SharePoint or keep accessible via Entra Private Access.

File share access options
  • Entra Private Access - ZTNA access to on-prem shares (recommended)
  • Azure Network Connection - Direct VNet connectivity
  • Entra ID Kerberos - SSO to file servers
Additional complexity

Traditional file shares require network connectivity from Cloud PCs. Consider migrating to OneDrive/SharePoint for better experience.

Migration tools

Use SharePoint Migration Tool or Migration Manager to move file shares to SharePoint/OneDrive.
