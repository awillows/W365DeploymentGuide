License Selection

Which Windows 365 license model fits your needs?

Windows 365 Enterprise

Recommended
  • Full Intune integration
  • Azure Network Connection support
  • Custom images
  • Granular provisioning policies
  • Hybrid Entra ID join option

Best for: Organizations with existing Microsoft 365 E3/E5 or EMS

Windows 365 Business

  • Simple setup via windows365.microsoft.com
  • No Azure subscription required
  • Self-service for users
  • Limited to 300 users
  • Entra ID Join only
  • Microsoft Hosted Network only
  • Gallery images only

Best for: Small businesses, quick pilots

Windows 365 Frontline

  • Shared Cloud PC model
  • Cost-effective for shift workers
  • 3 users per license
  • Full Enterprise features
  • Concurrent use limits

Best for: Shift workers, shared workstations

Identity Configuration

How should Cloud PCs join your directory?

Microsoft Entra ID Join

Recommended
  • Cloud-native
  • No on-prem infra
  • Simple setup
  • Works with MHN
  • SSO to cloud apps
  • Kerberos support*

Hybrid Entra ID Join

  • On-prem resources
  • Native Kerberos
  • Group Policy
  • Requires ANC
  • Complex setup
  • DC connectivity

Quick Decision Matrix

Scenario Recommendation
Cloud-first org Entra ID
File shares (SMB) Either*
Legacy NTLM apps Hybrid
Group Policy required Hybrid
Remote workers Entra ID

Key Considerations

Select an identity option

Choose an identity configuration to see relevant considerations and guidance.

Recommended Approach

Microsoft Entra ID Join is the modern, cloud-native approach and recommended for most deployments.

On-prem access? You're covered!

Entra ID Kerberos enables SSO to on-premises file shares and apps - no Hybrid Join needed!

  • Access file servers via SMB with Kerberos
  • No DC line-of-sight from Cloud PC required
  • Works with AD Connect + password hash sync
Learn more
Simpler, faster deployment

No domain controller connectivity needed. Works with Microsoft Hosted Network for zero infrastructure setup.

Better security posture

Cloud-native identity with Conditional Access, passwordless auth, and no legacy protocols.

Additional complexity

Hybrid Join adds significant infrastructure requirements. Consider if Entra ID Join with Kerberos could meet your needs instead.

Infrastructure requirements
  • Azure Network Connection (Azure subscription required)
  • VPN Gateway or ExpressRoute to on-premises
  • Domain Controller connectivity from Cloud PCs
  • Azure AD Connect with device writeback
Longer provisioning time

Domain join adds time to provisioning. Failed joins require troubleshooting DC connectivity.

When Hybrid is the right choice
  • Legacy apps requiring NTLM authentication
  • Strict Group Policy requirements
  • Regulatory compliance mandating domain join

Key Considerations

Select a network option

Choose a network configuration to see relevant considerations and guidance.

Recommended Approach

Microsoft Hosted Network is the simplest, fastest path to Windows 365. Zero infrastructure to manage.

Need on-prem access? Use Global Secure Access!

Entra Private Access provides VPN-less connectivity to on-premises resources - no Azure Network Connection needed!

  • Zero Trust Network Access (ZTNA)
  • Per-app Conditional Access policies
  • Lightweight connector on-premises
  • No inbound firewall rules
Learn about Private Access
No additional Azure costs

No VNet, VPN Gateway, or ExpressRoute required. Microsoft manages all networking infrastructure.

Entra Internet Access for security

Add cloud-native Secure Web Gateway - web filtering, threat protection, M365 optimization without on-prem proxies.

Learn about Internet Access
RDP Shortpath supported

Enable public RDP Shortpath for direct UDP connectivity and reduced latency.

Additional cost and complexity

ANC requires Azure infrastructure and ongoing management. Consider if Microsoft Hosted Network with Global Secure Access could meet your needs.

Azure costs to consider
  • VPN Gateway: ~$140-350/month (depending on SKU)
  • ExpressRoute: $55-500+/month + circuit costs
  • Data egress charges for on-prem traffic
  • VNet itself is free, but peering has costs
Management overhead
  • Subnet IP address planning (/26 minimum)
  • DNS configuration and troubleshooting
  • VPN Gateway maintenance and monitoring
  • Network security group rules
  • ANC health check failures to diagnose
Longer setup time

VPN Gateway deployment takes 30-45 minutes. ExpressRoute provisioning can take days. ANC validation adds to provisioning time.

When ANC is the right choice
  • Hybrid Entra ID Join requirement
  • High-bandwidth on-prem data access
  • Regulatory requirement for VNet isolation
  • Existing Azure infrastructure to leverage

Image Selection

What image will your Cloud PCs use?

Gallery Image

Quick Start
  • Pre-configured by Microsoft
  • Always up to date
  • Windows 11 + Microsoft 365 Apps
  • Multiple configurations available
  • No image management overhead
Available Images:
  • Windows 11 Enterprise + Microsoft 365 Apps
  • Windows 11 Enterprise
  • Windows 10 Enterprise + Microsoft 365 Apps
  • Windows 10 Enterprise

Custom Image

  • Pre-install LOB applications
  • Standardized configuration
  • Faster provisioning (apps pre-installed)
  • Requires image management
  • Enterprise license only
Requirements:
  • Azure subscription with Compute Gallery
  • Gen 2 VM image
  • Sysprep generalized
  • Windows 10/11 Enterprise

Key Considerations

Select an image type

Choose an image strategy to see relevant considerations and guidance.

Recommended Approach

Gallery images are the fastest path to deployment. Microsoft maintains them with the latest updates and security patches.

Apps? Use Intune!

Don't bake apps into images - deploy them via Intune instead:

  • Centralized app management
  • Easy updates without rebuilding images
  • User self-service via Company Portal
  • Works with any gallery image
Provisioning time

Gallery images provision quickly (~30-45 min). Apps deployed via Intune install during or after provisioning.

Always current

Microsoft updates gallery images monthly with latest Windows updates - no maintenance required on your part.

Additional overhead

Custom images require ongoing maintenance. Consider if Intune app deployment could meet your needs instead.

Maintenance requirements
  • Monthly image updates recommended
  • Sysprep and generalize for each update
  • Upload to Azure Compute Gallery
  • Test before production deployment
When custom images make sense
  • Very large applications (10GB+ like AutoCAD, Visual Studio)
  • Complex app dependencies requiring specific install order
  • Strict compliance requiring validated base images
  • Specialized developer environments
Hybrid approach

Put large/complex apps in the image, use Intune for everything else. This balances provisioning speed with manageability.

How will Windows updates be managed on your Cloud PCs?

Windows Autopatch

Recommended
  • Fully automated updates
  • Microsoft managed rings
  • Automatic rollback
  • Quality & feature updates
  • Microsoft 365 Apps updates
  • Driver updates

Best for: Hands-off update management

Windows Update for Business

  • Granular deferral controls
  • Custom update rings
  • Deadline enforcement
  • Intune policy-based
  • More management overhead
  • Manual monitoring required

Best for: Custom update schedules

WSUS / Legacy

  • Full approval control
  • On-premises infrastructure
  • Significant management overhead
  • Requires ANC
  • Legacy approach
  • Not recommended for Cloud PCs

Best for: Existing WSUS infrastructure (migration path)

Key Considerations

Select an update strategy

Choose how Windows updates will be managed on your Cloud PCs.

Recommended Approach

Windows Autopatch provides hands-off, enterprise-grade update management. Microsoft handles the complexity.

What Autopatch manages
  • Windows quality updates (monthly)
  • Windows feature updates (annual)
  • Microsoft 365 Apps updates
  • Microsoft Edge updates
  • Microsoft Teams updates
Deployment rings

Autopatch automatically creates deployment rings (Test First Fast Broad) to safely roll out updates with automatic rollback on issues.

Licensing

Included with Windows E3/E5 or Microsoft 365 E3/E5 licenses. No additional cost.

Learn about Autopatch
More control, more work

WUfB gives you granular control but requires you to create and manage update policies in Intune.

Configuration options
  • Feature update deferral (0-365 days)
  • Quality update deferral (0-30 days)
  • Deadline enforcement
  • Active hours configuration
  • Update rings for phased rollout
Not recommended for Cloud PCs

WSUS requires Azure Network Connection and adds significant management overhead. Consider migrating to Autopatch or WUfB.

How will applications be deployed to your Cloud PCs?

Microsoft Intune

Recommended
  • Cloud-native deployment
  • Win32 apps, MSI, MSIX
  • Microsoft Store apps
  • Microsoft 365 Apps
  • Required & available modes
  • App protection policies

Best for: Modern app management

Intune + Custom Image

  • Large apps pre-installed
  • Faster user onboarding
  • Intune for updates
  • Image maintenance
  • Enterprise only

Best for: Large LOB apps (CAD, etc.)

SCCM / Legacy

  • Existing infrastructure
  • Complex deployments
  • Requires ANC
  • On-prem infrastructure
  • Higher complexity

Best for: Existing SCCM customers (co-management)

Key Considerations

Select an app deployment strategy

Choose how applications will be delivered to your Cloud PCs.

Recommended Approach

Intune provides cloud-native app deployment that works seamlessly with Windows 365 regardless of network configuration.

Supported app types
  • Win32 apps - .exe, .msi with detection rules
  • Microsoft Store - New Store experience
  • Microsoft 365 Apps - Built-in deployment
  • LOB apps - .msi, .appx, .msix
  • Web apps - PWA/shortcuts
Company Portal

Users can self-service install available apps from Company Portal - reducing IT tickets and improving productivity.

Hybrid approach

Pre-install large applications (AutoCAD, Visual Studio, etc.) in the image, then use Intune for updates and smaller apps.

Image maintenance required

You'll need to update and re-upload the image periodically to include app updates. Consider if Intune-only would simplify operations.

Additional complexity

SCCM requires Azure Network Connection for Cloud PCs to reach distribution points. Consider co-management as a path to Intune.

Co-management path

Use co-management to gradually shift workloads from SCCM to Intune. Start with compliance policies, then apps, then updates.

User Data & Storage

Where will users store their data?

OneDrive for Business

Recommended
  • Known Folder Move
  • Cross-device sync
  • Files On-Demand
  • Version history
  • Ransomware recovery
  • No VPN needed

Best for: Cloud-first data strategy

Hybrid (OneDrive + File Shares)

  • OneDrive for user data
  • File shares for team data
  • Gradual migration path
  • May need ANC or GSA
  • Mixed experience

Best for: Transitioning to cloud

Traditional File Shares

  • Existing infrastructure
  • Familiar UX
  • Requires network path
  • No offline access
  • Backup complexity
  • Data not portable

Best for: Regulatory requirements

Key Considerations

Select a data storage strategy

Choose where users will store their documents and files.

Recommended Approach

OneDrive with Known Folder Move provides the best user experience for Cloud PCs - data follows the user everywhere.

Known Folder Move (KFM)

Automatically redirect Desktop, Documents, and Pictures to OneDrive:

  • Seamless user experience - no change in behavior
  • Data available on any device
  • Automatic backup and sync
  • Configure via Intune policy
Configure KFM
Files On-Demand

Files appear in File Explorer but download on access - saving Cloud PC disk space while providing full access to user's OneDrive.

Built-in protection

OneDrive includes version history (30 days), ransomware detection, and file recovery. Data is protected even if Cloud PC is compromised.

Transition strategy

Start with OneDrive for user profiles, gradually migrate team shares to SharePoint or keep accessible via Entra Private Access.

File share access options
  • Entra Private Access - ZTNA access to on-prem shares (recommended)
  • Azure Network Connection - Direct VNet connectivity
  • Entra ID Kerberos - SSO to file servers
Additional complexity

Traditional file shares require network connectivity from Cloud PCs. Consider migrating to OneDrive/SharePoint for better experience.

Migration tools

Use SharePoint Migration Tool or Migration Manager to move file shares to SharePoint/OneDrive.
